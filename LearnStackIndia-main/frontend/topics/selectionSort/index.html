Selection Sort: An In-Depth Look

Understanding the Mechanics of Selection

Beginner Friendly

üí° Fundamental Principle

Selection Sort works by dividing the input list into two parts: a sorted sublist at the beginning and an unsorted sublist at the end. Initially, the sorted sublist is empty. The algorithm repeatedly finds the minimum element (for ascending order) from the unsorted sublist and swaps it with the first element of the unsorted sublist. This moves the boundary between the sorted and unsorted sublists one element to the right.

64 34 12 22 ‚Üí Swap with 1st unsorted (64) ‚Üí 12 34 64 22
Pass 1: Find minimum (12) ‚Üí Swap with first element (64)

üîÑ Algorithm Flow

1 Assume the first element of the unsorted part is the minimum.
2 Iterate through the rest of the unsorted part, comparing each element with the current minimum.
3 If a smaller element is found, update the index of the minimum.
4 After scanning the unsorted part, swap the found minimum with the first element of the unsorted part.
5 Repeat until the entire list is sorted.

üìä Performance Analysis

Selection Sort's time complexity is consistent regardless of the initial order:

Best Case Time O(n¬≤)
Still requires nested loops even if sorted.
Average Case Time O(n¬≤)
Typical quadratic performance.
Worst Case Time O(n¬≤)
Occurs for any input order.
Space Complexity O(1)
It's an in-place algorithm.
Swaps O(n)
Performs at most n-1 swaps, useful when swaps are costly.

// Selection Sort in JavaScript
function selectionSort(arr) {
  const n = arr.length;
  for (let i = 0; i < n - 1; i++) {
    let minIndex = i;
    for (let j = i + 1; j < n; j++) {
      if (arr[j] < arr[minIndex]) {
        minIndex = j;
      }
    }
    if (minIndex !== i) {
      [arr[i], arr[minIndex]] = [arr[minIndex], arr[i]]; // Swap
    }
  }
  return arr;
}

const arr = [64, 25, 12, 22, 11];
console.log(selectionSort(arr));

def selection_sort(arr):
  n = len(arr)
  for i in range(n - 1):
    min_index = i
    for j in range(i + 1, n):
      if arr[j] < arr[min_index]:
        min_index = j
    if min_index != i:
      arr[i], arr[min_index] = arr[min_index], arr[i] # Swap

// Selection Sort in Java
public class SelectionSort {
  static void selectionSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
      int minIndex = i;
      for (int j = i + 1; j < n; j++) {
        if (arr[j] < arr[minIndex]) {
          minIndex = j;
        }
      }
      if (minIndex != i) {
        int temp = arr[i];
        arr[i] = arr[minIndex];
        arr[minIndex] = temp; // Swap
      }
    }
  }

  public static void main(String[] args) {
    int[] arr = {64, 25, 12, 22, 11};
    selectionSort(arr);
    for (int x : arr) System.out.print(x + " ");
    System.out.println();
  }
}

#include <stdio.h>

void swap(int *xp, int *yp) {
    int temp = *xp;
    *xp = *yp;
    *yp = temp;
}

void selectionSort(int arr[], int n) {
    int i, j, min_idx;
    for (i = 0; i < n-1; i++) {
        min_idx = i;
        for (j = i+1; j < n; j++)
          if (arr[j] < arr[min_idx])
            min_idx = j;

        if(min_idx != i)
            swap(&arr[min_idx], &arr[i]);
    }
}

    for (int i=0; i < size; i++) printf("%d ", arr[i]);
    printf("\n");
}

‚ÜîÔ∏è Stability

Standard implementations of Selection Sort are generally not stable. This means that if there are elements with equal values, their relative order might change after sorting. For example, if you have `[5a, 3, 5b, 1]` (where `5a` and `5b` are equal but distinct), Selection Sort might result in `[1, 3, 5b, 5a]`, changing the original order of the 5s. While stable versions exist, they often require extra space or complexity.

üîë Key Characteristics

  • Simplicity: Easy to understand and implement.
  • In-Place: Requires minimal extra memory (O(1) space).
  • Minimal Swaps: Performs O(n) swaps, which can be advantageous if writing to memory is expensive.
  • Inefficient Time: O(n¬≤) time complexity makes it unsuitable for large datasets.
  • Not Adaptive: Performance doesn't improve significantly for already sorted or nearly sorted data.
  • Not Stable: Standard implementation doesn't preserve the relative order of equal elements.
